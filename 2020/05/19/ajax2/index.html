ajax实战

初学ajax把ajax的大概功能都了解了，接下来就根据实验三来试试ajax的具体应用

实验三的detail页面根据实验报告还有直播有两种不同的应用

第一种，使用form提交到servlet页面，页面跳转，这一种是不能使用ajax的

第二种，使用ajax提交

刚尝试ajax的时候，我没有把form改掉，依旧使用了form提交，然后页面就会跳转，出现一句话 isSuccess：true,msg:加入购物车完成;我还很惊奇怎么都是true，页面还会跳转。所以一定要记好ajax就是直接传递数据，页面不会跳转。 


$(document).ready(function() {
     $(".btn btn-default").click(function() {
     var quantity = parseInt($(".shuliang").val());
     var id = $(".single-right").data("id");
     //通知服务端修改Session中购物车内容
       $.ajax({
        type : "post",
        url : "AddToCartServlet2",
        data : {
              "id" : id,
              "quantity" : quantity
               },
        success : function(data) {
                     //当服务端正常运行完成，并返回json数据时，修改界面上的内容
           if (data.isSuccess == "true") {
                       //修改购物车 显示数量
               var totalQuantity = parseInt($(".quantity").text());
               $(".quantity").text(totalQuantity + quantity);                           
                   toastr.success("商品已加入购物袋");
                      } else {
                    toastr.error("加入购物袋失败：" + data.msg);
                         }
                 }
          });
      });
  });

这里注意使用ajax就需要使用 JsonResult 


        public class JsonResult {
        private boolean success;
        private String msg;

        @Override
        public String toString() {
            return String.format
                    ("{\"isSuccess\":\"%s\",\"msg\":\"%s\"}",
                            getSuccess(),getMsg());
        }

通过issuccess，msg显示的数据，可以判断出商品数量是否出现问题，是否成功添加

cart页面也使用了ajax，依旧是基本的框架


$('.close1').click(function () {
     id = $(this).parent().attr("id");
     $.ajax({
         url: "RemoveFromCartServlet",
         data:{ "id": id},
         success:function (result) {
         toastr.success(result);
         result = JSON.parse(result);
         if (!result.isSuccess){
                  window.location.href = "index.jsp";        
            }else {
              window.location.href = "cart.jsp";
                  }
          }
    });

ajax的格式都是大致不变的，核心的难点就是servlet，数据判断、封装进对象、迭代器计数、数据写入对象都是根据不同的要求有不同的写法

基本的格式应用都是相同的，把不同功能所需要的不同步骤细分，掌握好他独有的步骤,就可以解决很多很多问题，当然，可能需要一次次的摸索，实验三很难，他需要花费很长的时间去摸索不同功能的解决方法，凡是一旦解决，就会收获满满的快乐。
